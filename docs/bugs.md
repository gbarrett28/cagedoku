Things that are wrong:

1. User still has control of the essential highlight.  Clicking on a candidate cycles essential -> impossible -> possible.  This needs to be updated to cycle between possible/impossible.  The essential highlight should be determined automatically from the solutions.
2. We need a config option to enable/disable the essential highlight.
3. We need to recognise the grid type from the image rather than specify it in a drop-down.  Sooner or later, we need to try to recognise formats we have not seen before.
4. The undo feature is not working.
5. Rule triggers do not appear to be sensitive to user changes to candidate sets.
6. Some of the rules do not feel pure enough.  Eg, it seems like some narrow solution sets when they are not narrowed in the candidate view.
